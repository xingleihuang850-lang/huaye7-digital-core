## B2-min Definition

B2-min is a minimal calibrated continuation, not a large new training program.

It should answer only:

1. Can the B1.1 conditional-pass calibrated gray/pore pipeline be used as a stable starting point for B2?
2. Does adding the next B2 component improve robustness without breaking B1.1 topology gates?
3. Is the calibrated pipeline reproducible with manifests and hashes?

Out of scope for B2-min:

- 100/200 epoch scaling.
- Second topology rescue.
- Uncalibrated ORIG raw claims.
- Full B2 production training.

## Proposed B2-min Variants

### Variant A: Calibrated post-selection baseline

Use existing `ep015` generated gray samples, apply qmatch, then choose/record threshold/nnUNet masks under fixed rules.

Purpose: establish B2-min baseline with no new training.

Inputs:

- `/home/user/HXL/HY7_planb/phase2/b1_gray_sus_b11_rescue20_formal512_20260706/ep015/samples_gray.npy`
- `/home/user/HXL/HY7_planb/phase2/slices_ct28_gray_128/test.npy`
- `/home/user/HXL/HY7_planb/phase2/slices_ct28_gray_128/test_pore.npy`

Output directory:

- `/home/user/HXL/HY7_planb/phase2/b2_min_calibrated_baseline_ep015_20260706`

Gate:

- qmatch manifest present.
- formal metrics pass: S₂≤0.003, Euler≥115, maxCC≤0.070.
- nnUNet qmatch reverse_fail=False.

### Variant B: Calibrated constrained selection

Generate or reuse multiple ep015 sample batches, apply qmatch, then select candidates by a fixed, pre-registered score:

`score = z(S2_rmse) + z(abs(Euler-120)) + z(maxCC)`

No topology proxy retraining.

Purpose: test whether selection improves stability over one batch without changing the model.

Gate:

- Selection rules pre-registered before running.
- No threshold/gate relaxation.
- Must report selected and rejected candidates.

### Variant C: Minimal calibrated B2-min handoff package

Package the calibrated ep015 candidate and all required manifests/hashes as the input bundle for later B2.

Purpose: make B2-min reproducible before any heavier B2 experiment.

Gate:

- Bundle includes checkpoint hash, qmatch manifest, formal metrics, nnUNet review metrics, and ORIG raw known-fail note.
