## 0. 论文 T0613 = 开题分割的"最新进展"，它的留白就是花页7 的入口

论文做完且**定量**的（`[文]`）：
- 自建 SEM(0.3μm)+QEMSCAN 六类（孔隙/有机质/粘土/长英质/碳酸盐/金属重矿物）2D 分割：mIoU 62.34 / mDice 74.61（nnU-Netv2）；**UNet3+ 反而最高**(63.07/75.74)，nnU-Net 略低但自动化最强。
- 阈值分割对照：**孔隙 IoU/Dice=100%**、有机质也高，但碳酸盐 17.0%、金属重矿物 14.9% → 灰度突出类阈值就够，灰度重叠类必须 AI。
- 外部泛化只到 **2 类**(cigRockSEM pore IoU 56.18) 和 **3 类**(MudrockNet clay/silt/pore，mIoU 79.75)——**因为公开页岩集只标到这。**

论文**没做/只定性**的（= 花页7 的贡献空间）：
- §2.4 0.3μm CT、§2.5 4nm CT 跨模态/跨尺度迁移 **无像素级真值 → 只能定性**，算不出 Dice/IoU；结论明写"后续需补带精确标注的 CT/FIB-SEM/QEMSCAN 配准数据，建立严格跨模态定量验证"。
- 全部是 **2D**；3D 仅作可视化重建展示，非定量。
- 阈值与 AI 是**并列对比**，没有"阈值×AI 混合"。

→ **花页7 恰好补这三个洞**：配准 CT 体 + 现成 3D Avizo 标签 + Amics 2D 矿物 GT，使 (a) 多矿物能在独立井定量验证、(b) CT 跨模态能定量、(c) 真 3D 分割可做。**中期汇报的主张就立在这。**

## 1. 回答"为什么 E1/E2/E3 只有 孔隙/裂缝/基质，不涉及矿物？" `[测]`

**不是我随意选三相，是 14μm CT 这个尺度服务商 Avizo 只提供了 sus/pore/feng 三个组件体。** 各尺度服务商给的标签（核盘结果）：

| 尺度 | 维度 | 相标签 | → 能做 |
|---|---|---|---|
| 14μm CT | 真3D | sus+pore+feng | 三相(基质/孔隙/裂缝) ← E0/E1/E3 用的就是这个 |
| 2.8μm CT | 真3D | sus+pore | 两相(无裂缝) ← E2 |
| 65nm / FIB-SEM | 真3D | sus+pore+youjizhi+feng | 四相(含有机质) |
| 10nm Maps | **2D** | 无机孔/有机孔/无机缝/有机缝 | 有机无机区分，但2D |
| 25μm/1μm Amics | **2D** | 矿物图(BSE+Mineral) | 多矿物，但2D |

要点：
- **矿物身份在粗 CT 灰度里拿不到**（CT 测的是衰减≈密度×Z，长英质/碳酸盐/粘土大量重叠），矿物只能来自 2D 的 Amics/Maps。所以三相 CT 分割与多矿物分割是**两条不同模态的线**，不是一次能合做的任务。
- **有机孔/无机孔**：真 3D 体里**没有直接标签**；只有 10nm Maps 在 2D 上分了。但 65nm/FIB-SEM 有 `pore`+`youjizhi` 两相，**可推导**：有机孔≈pore∩有机质内，无机孔≈pore 在矿物基质——这是 3D 有机/无机孔的唯一可行来源（`[假]`待验证）。
- 用户直觉对：**分割对象越多越难，先 2→3 相打牢**，矿物/有机孔分级是后续逐步加的，不强求一次全分准。

## 2. 裂缝为什么难 / 阈值 vs AI vs 混合 `[测][文]`

- **裂缝难**：CT 里裂缝与孔隙**灰度都≈0**（都是低密度空腔），灰度上分不开；区别是**几何**——裂缝是平面/贯通的薄片状，孔隙近等轴。所以裂缝-孔隙分离是**形态/连通性问题**，不是灰度问题，这正是 E0 裂缝 dice 只 0.57（孔隙 0.92）的根因，也是 E1 加权未显著收敛的原因。→ 裂缝单独成线，可引形态学/方向滤波/3D 连通性先验。
- **三方法对比（用户提议，且论文数据已背书）**：
  1. **纯阈值**（传统）：灰度突出相（孔隙/有机质）几乎免费且准（论文孔隙 100%）。
  2. **纯 nnU-Netv2**：灰度重叠相（矿物）必须靠它。
  3. **阈值×AI 混合**（论文**没做**，空白点）：先阈值切出无灰度重叠的相（孔隙/有机质/黄铁矿亮点），从图像中扣除/作为先验通道，剩余灰度重叠区再交 nnU-Net → **降类别数、降不平衡、可能更准**。
  - 设计成**受控三臂对照**：同数据同划分，比 IoU/Dice/mIoU + 推理耗时 + 人工量。先在 2 相、3 相做，再上多矿物。这本身可成中期汇报一个**方法贡献点**。

## 3. 2D 分割路线（中期汇报主体，逐级加难度）

| 编号 | 实验 | 数据 | 锚点 |
|---|---|---|---|
| **S1** | 三方法对照（阈值/nnU-Net/混合），**2 相**(基质/孔隙) | Amics BSE 或 CT 切片 | 复刻+扩展论文§2.1，加混合臂 |
| **S2** | 同上，**3 相**(+裂缝 或 +有机质) | 同 | 验证混合在多类的增益 |
| **S3** | **Amics 多矿物**(对齐论文六类口径) | Amics 25μm BSE→Mineral 配对 | **补论文缺的"独立井多矿物定量验证"** |
| **S4** | **跨地区融合鲁棒性**：花页7 ∪ 官页15-1-1（统一类标） | 两井 SEM/Amics | 用户"多地区数据集→更普适、不偏训练集" |

- **S3 关键**：Amics 25μm 的 `BSE_Image` 与 `Mineral_Image` 同尺寸=像素对齐，是现成 (灰度,矿物标签) 配对，直接做成 `Dataset7xx`。先核**矿物色板→类别值映射**（对齐论文表1六类合并口径），再切 256×256，nnU-Netv2 2D。
- **S4 融合**：注意域差（不同井灰度/制样差异）→ 可做"单井训→他井测"的跨域评估，量化鲁棒性提升；这正面回应"不偏向训练数据"。

### 3b. "能不能改 nnU-Netv2 的医学默认参数？" → 能，且有官方机制 `[文][核]`
- nnU-Net **本来就按数据指纹自配置**（中位形状/体素间距/强度分布→patch/网络深/归一化），不是死搬医学参数——论文§1.3.4 已述。
- 仍想"灵活改"的合法入口（v2 原生支持，不破坏自配置）：
  - **自定义 Trainer**（`nnUNetTrainer` 子类）：改 epoch/损失权重/类别重采样/数据增强（地学纹理 ≠ 医学，增强策略可调）。
  - **自定义 Plans / ResEnc 预设**：改 patch、batch、归一化（CT 用 `CTNormalization`，SEM 用 `ZScore`）。
  - **少数类增强**：论文结论自己建议——对碳酸盐/金属重矿物加 loss 加权、类别重采样、先验约束。这是 S3 直接可落的改造点。
- → 写汇报时表述为"**在 nnU-Net 自配置框架内做面向页岩的 Trainer/Plans 定制**"，而非"推翻其参数"。

## 4. 3D 分割路线（用户真正想要的 nnU-Net 3D 能力）`[测]`

- **已证可行**：E2 自写3D UNet 0.939、E3 nnU-Net 3d 0.919 / 2d 0.966（同网格）。配准修复后 3D 孔隙/三相**不是问题**。
- **S5 — 3D 孔隙/三相，用 Avizo 标签做 nnU-Net `Dataset7xx`**：14μm(三相)、2.8μm(两相)、65nm/FIB-SEM(四相)。**与服务商 Avizo 原结果对比**（Dice/IoU + 孔隙度% 对账），看 nnU-Net 能否复现并"学到 3D 连通信息"（z 向贯通是自写UNet/2D 学不到的）。
- **S6 — 3D 有机孔/无机孔**：真 3D 无直接标签 → 用 65nm/FIB-SEM 的 `pore∩youjizhi` 推导有机孔、其余为无机孔，作弱标签训练；与文献有机/无机孔表征（蔡潇2020）口径对齐。诚实标注这是**推导标签**非服务商直给。
- **S7 — 3D 矿物 = 跨模态推断（非纯分割）**：CT 灰度无法唯一定矿物 → 把 Amics 2D 矿物图配准到邻近 CT 切片学映射，沿体推断 3D 矿物场，**必带不确定性**。这其实已迈入生成式（见 §6），不要包装成"nnU-Net 直接出 3D 六矿物"。

## 5. 多尺度：升尺度矿物预测 + 内存 / "压缩感知" `[文][假]`

用户想法：从 Amics(细) 学矿物 → 升尺度到粗分辨率矿物预测；但加细分辨率信息→内存爆 → 借鉴医学压缩感知。澄清与优化：
- **维度/方向**：Amics 是**细尺度 2D**，"升尺度到粗"= 把细尺度学到的矿物-纹理关系迁移到粗 CT。这是**跨尺度迁移/超分的逆问题**，对应文献 [[02_文献综述]] C 组(超分) + 王玥天2025(数字岩心多尺度多组分融合)。
- **内存大的对口解**（先用成熟的，别先上 CS）：
  - nnU-Net **3d 级联**(3d_lowres→3d_fullres)——论文§1.3.4 正是为"全分辨率大体难训"设计；
  - patch 滑窗 + 混合精度 + 梯度检查点（已在用 AMP）。
- **"压缩感知"定位要诚实**：医学 CS 主要用于 **MRI 采集加速/欠采样重建**，不是分割省内存。但其**稀疏先验思想**可类比"**从稀疏 2D 矿物切片恢复 3D 矿物场**"——那是**重建/生成**问题(§6)，不是分割。→ 查文献时按"sparse-view / limited-angle CT 重建""implicit neural representation 3D rock""cross-scale super-resolution digital rock"检索，而非"分割+压缩感知"。

## 6. 衔接数字井筒（分割做好之后）`[假]`

用户两条路，关系是**先对比、可收敛为结合**：
- **A 路 分割填充**：先建 孔隙+基质 两相骨架（裂缝因复杂暂缓，待 GJ5-15 电成像给裂缝标签再加）→ 骨架向井深延伸 → 用 §3/§4 的分割器往里填矿物。
- **B 路 直接生成**：生成式模型（GAN/扩散，文献 B/C/D 组）直接生成孔隙/裂缝/矿物。
- **建议**：先各自做、对比（参数：孔隙度/连通性/矿物比例 vs 实测），再**结合**——B 生成结构、A 的分割器作"判别器/物理校正"约束（呼应 Wang2021b "提升多矿物分割物理精度"）。
- **物理意义**（生成随机性的缰绳）：阶段四用 GJ5-15 测井/电成像，Archie 电性正演 + 核磁正演 → **模拟测井 vs 实测**作为生成数字井筒的"可用性"判据。与路线图 D2/D3 一致：**放后期、做验证手段，不喧宾夺主**。

## 7. 中期汇报故事线（一句话串起来）

> 开题分割停在 **2D、单井、矿物只能内部验证(论文 T0613)** → 花页7 提供**配准 CT 体 + 3D Avizo 标签 + Amics 2D 矿物 GT** →
> ① 证 nnU-Netv2 在花页**真数据**可行(E2/E3，配准修复 0→0.92，模型变量排除)；
> ② **阈值×AI 混合**新方法，2→3 相→多矿物逐级；
> ③ 把论文只能定性的 **CT 跨模态/3D** 做成**定量**(S3/S5)；
> ④ 跨地区融合(花页∪官页)证鲁棒 → 为阶段二(2D 扩散)/三(多模态 3D)/四(数字井筒)造**可信三维相场底料**。
