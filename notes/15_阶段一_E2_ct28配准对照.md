# E2：ct28 Plan B baseline —— 配准修复决定性对照

> 主线价值：同一口井、同一尺度(2.8μm)、同一任务(孔隙分割)，**只改图像来源**——
> codex 用 2024³ 原图+裁剪 → dice≈0；我用同网格 sus+pore==0 → 孔隙 dice 0.939。
> 证明 codex 失败的病根是**配准（裁剪错位）**，Plan B 同网格直接根治。这是中期汇报/论文最干净的一张对照。
>
> **P4 边界补记（2026-07-12）**：E2 训练和固定验证 patch 都来自同一 ct28 体，未保存 patch 坐标或空间 split manifest。因此 `0.939` 是同体内部 patch-validation 指标，不能写成空间留出、独立体或跨井泛化；E2 仍支持同网格 formulation 在该体内可行。

## 科学判读（含诚实局限）

- **结论**：在这组不同工程/评估设置的对照中，dice 从 ≈0 跃到 0.94，最关键且已证实的变化是**图像-标签是否同网格共配准**。
  这与 codex 自己 6/22 的判断一致（"改用 sus+pore==0、弃 2024³ 裁剪"），E2 是该判断的**实证确认**。
- **局限（不掩盖）**：codex 用 nnU-Netv2、我用自写 UNet，模型不同，是一个混淆因素。但：
  - 0→0.94 的量级**无法用模型差异解释**（nnU-Netv2 很强，well-posed 问题不会得 0）；得 0 是**监督被错位破坏**的症状。
  - 要完全隔离模型变量 → **E3：用 nnU-Netv2 在同网格 sus+pore 上再跑一版**（codex 的工具 + 我的数据选择）。预期 nnU-Netv2 也将获高 dice，从而把"是配准不是模型"钉死。
- 报告孔隙度仍用 codex 的精确关系（面孔率%≈1.290×raw方形pore==0比例），E2 是分割质量、非孔隙度复算。
