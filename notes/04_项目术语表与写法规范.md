# 项目术语表与写法规范

> 目的：统一全项目文档中的模型名、实验名、指标名、路径口径与状态表达，避免同一概念多种写法。  
> 使用规则：新写或修改任何文档前，先按本表检查；历史笔记中如果是原始证据/stdout/命令/文件名，可保留原样并在旁边解释。

## 1. 模型与方法名称

| 统一写法 | 不推荐写法 | 使用说明 |
|---|---|---|
| `nnU-Netv2` | `nnU-Net v2`、`nnU-Net`、`nnUNet`、`nnunet`（作为模型名时） | 作为模型/框架名称时统一写 `nnU-Netv2`。 |
| `U-Net` | `UNet`（作为论文模型名时） | 标准模型名；但代码类名/文件名中的 `UNet` 保留。 |
| `3D U-Net` | `3D UNet`（作为论文模型名时） | 论文/模型名统一加连字符；代码名可保留。 |
| `DDPM` | `ddpm`（作为方法名时） | 方法名大写；目录、脚本、命令中的小写 `ddpm_*` 保留。 |
| `diffusers` | `Diffusers` | Python 包名保留小写。 |
| `M7-v2` / `M7-v3` | `M7v2` / `M7v3` | 实验版本统一加连字符。 |
| `200ep` | `200 epoch`、`200Epoch` | 本项目实验简称统一 `50ep`、`200ep`。 |

### 例外

以下内容按原样保留，不强行改：

- 文件名、目录名、脚本名、环境名、CLI 命令，例如：
  - `nnunet_pipeline`
  - `nnunet_t28`
  - `nnUNet_raw`
  - `nnUNetv2_train`
  - `src/hy7_ct14_make_nnunet_3phase.py`
- 原始 stdout / hash 证据 / 远程命令输出。
- 已存在的 Obsidian 文件名，例如 `16_阶段一_E3_nnUNet同网格核验.md`，为避免破坏链接，文件名暂不重命名；正文标题和显示名可写 `nnU-Netv2`。

## 5. 状态词

状态词沿用 `notes/03_项目Workflow设计.md`：

| 统一状态 | 含义 |
|---|---|
| `planned` | 已设计，未执行。 |
| `ready` | 输入、路径、命令、门槛齐备，可执行。 |
| `running` | 正在执行。 |
| `remote_done_not_evaluated` | 远程训练/计算已完成，但尚未采样/评估/解释。 |
| `trained_needs_sampling_eval` | 模型训练完成，缺采样与评估。 |
| `evaluated_needs_review` | 已有评估结果，待解释/复核。 |
| `accepted` | 结果已通过门槛，可写入主线。 |
| `rejected_or_negative` | 负结果，保留证据但不作为主线推进。 |
| `blocked` | 缺数据、缺权限、环境失败或结论不可信。 |
| `deprecated` | 历史方案，不再作为主线使用。 |

## 6. 表述规范

### 6.1 生成与分割不要混写

统一表述：

```text
真实 sus 输入 → pore 输出 是分割，不是生成。
```

M7-v3 后续方向应写为：

```text
B1：生成连续灰度 sus 介质本身。
B2：联合生成 [sus,pore] 双通道。
```

### 6.2 200ep 状态不要夸大

统一表述：

```text
M7-v3 200ep 已训练完成，但尚未采样/评估；不能判断 S₂/Euler/连通性是否改善。下一步应对 200ep best.pt 采样并复用 eval/calib 口径。
```

### 6.3 一阶段与二阶段关系

统一表述：

```text
阶段一分割已达到启动阶段二生成的地基条件，但不是说分割绝对终结；二阶段异常时仍可回溯检查标签、配准和切片问题。
```

### 6.4 扩散意义

统一表述：

```text
扩散模型的意义不是单纯生成好看的图像，而是把有限、多尺度、离散的岩心扫描数据，转化为可补全、可外推、可表征不确定性的数字岩心/数字井筒生成模型。
```

## 7. 后续维护规则

1. 每次新增术语，先加到本文件，再写入正文。
2. 修改旧文档时，只改解释性正文；原始证据、命令、路径、stdout 不强行规范化。
3. 若发现同一概念有两种写法，优先改入口文档：`CLAUDE.md`、`notes/README.md`、`notes/00`、`notes/03`、本文件。
4. 大规模重命名文件前必须先征得用户确认。
