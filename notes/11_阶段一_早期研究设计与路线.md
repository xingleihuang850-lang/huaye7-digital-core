## 0. 研究主线与本工作的定位

- **主线（不可偏离）**：花页7 多尺度数字岩心 → **生成式数字井筒**。即从"几块扫描过的岩心"
  生成"沿井深连续的数字井筒"，贡献点是**生成式建模**（超分 / 补全 / 虚拟岩心 / 不确定性）。
- **判别式→生成式 的桥**：生成模型要有"干净的多尺度三维相场"做训练真值。
  因此**分割(判别式)不是目的，是给生成式造底料**：先把各尺度 CT 体可靠地切成
  基质/孔隙/裂缝三维相场，并把多尺度对齐，生成式才有东西可学。
- **本工作（Plan B, ct14 三相分割）在主线中的坐标**：
  `多尺度数据掌握 → [本工作: 对齐+三相分割] → 多尺度融合 → 生成式`。

## 2. 数据分析（花页7，已核校）

- 花页7 @ 4199.21 m 钙质长英质页岩；6 尺度 25μm→8.589nm（跨~3000 倍）。
- **多尺度孔隙度非单调** 1.56/7.18/1.93/0.41%（14μm/2.8μm/65nm/Maps）= 尺度效应/REV；
  FIB-SEM 1.52% 是**有机质占比**非孔隙度（已剔除）。→ 孔隙度不能简单相加，必须"对齐+融合"。
- 矿物：长英质 54.6% + 碳酸盐 41.3% + 黏土 2.9% + 黄铁矿 0.48%。
- 连通性弱：配位数集中低值、0 占最大（14μm 达 85%）。
- **本工作新增的数据事实**（[测]，见实验记录）：服务商「处理图像」的 sus/pore/feng 同网格、
  天然共配准（孔隙∩裂缝重叠=0%）；sus=灰度、相标记值=0（与你 Dataset712 的"pore 自值 0"一致）。

## 4. 模型选型与理由

**本工作（Plan B 三相分割）选了轻量自写 3D U-Net**，理由与权衡：

| 候选 | 取舍 |
|---|---|
| **自写 3D U-Net（已用）** | ✅ 能直读完整体素在线采样、快速迭代、原生多类(含裂缝)、依赖少。作为**快速基线**先打通主线。 |
| nnU-Netv2（你的主力） | 你既有管线，**论文级基线/可比性强**。建议后续把 ct14 三相也做成 `Dataset7xx` 用 nnU-Netv2 跑，与自写 UNet **交叉验证**（这正是 §5 的下一步之一）。 |
| 2D 逐切片 | 丢失 z 向连通性，对裂缝(三维贯通)不利 → 不作主选。 |
| Transformer/SwinUNETR | 数据量/算力性价比此阶段不占优，留作消融。 |

> 结论：**自写 3D UNet 做快速基线打通"对齐→三相"，再用 nnU-Netv2 复核**——既快又有论文级可比性。
> 与既往联系：分割沿用你的判别式路线与"pore=0"编码；**新意在多尺度对齐框架 + 为生成式造底料**。

**下一跳（生成式）候选**（待 §5、文献后定）：3D 超分(SR-GAN/扩散)、多尺度条件生成、
缺失补全(inpainting)、虚拟岩心序列(沿井深)。详见 [[02_文献综述]] 待补的"生成式数字岩心"专题。

## 5. 实验路线（为什么先做这组 + 接下来怎么走，全部锚定主线）

**为什么先做 ct14 三相对齐+分割（本组实验的目的）**：
1. **解锁多尺度对齐**：先证明"同网格组件体天然共配准"这条 Plan B 路走得通（已证：重叠 0%），
   否则后面"多尺度融合→生成"无从谈起。
2. **补齐相场**：你已有 ct28 2 相；补 ct14 的**裂缝相**，才有完整 {基质/孔隙/裂缝} 三维相场。
3. **造生成式的底料**：可靠三维相场 = 生成模型的训练真值。
4. **低风险快速验证**：自写 UNet 快速打通，确认数据/标签/管线没问题（已得 dice 0.831）。

**接下来（建议顺序，每步出一份编号报告 + 证据）**：

| # | 实验 | 目的（锚主线） | 产出 |
|---|---|---|---|
| E1 | ct14 三相**收敛化**（加 epoch / 裂缝加权 / **固定种子可复现**） | 把裂缝 dice 0.57→更高，得可信相场 | 模型+指标+推理叠加图 |

2. 测试真值绝不进训练；不伪造标签。
3. **所有结论必须由实际运行结果支撑**；数字标来源、分 `[测]/[核]/[假]`。
4. 阻塞如实报告，不包装；不做不公平比较。
5. 局限写明（如本工作训练未固定种子→非逐位复现、轴序仅视觉验证）。
